## üìñ Guide Structure

Each demo guide follows the same comprehensive structure:

### 1. Persona Overview
- Who they are (role, responsibilities, typical workday)
- What they care about (3-4 key goals with metrics)
- Pain points before V17 (3 specific problems)
- How V17 transforms their workflow (value proposition)

### 2. Demo Scenarios (5 categories per persona)
- **Daily Operations**: Morning routines, priority checking (6-7 scenarios)
- **Domain-Specific Work**: Role-specific tasks (6-7 scenarios)
- **Communication/Collaboration**: Team coordination, stakeholder updates (5-7 scenarios)
- **Reporting & Analysis**: Executive summaries, trend analysis (5 scenarios)
- **Risk/Issue Management**: Blockers, escalations, critical issues (5 scenarios)

### 3. Scenario Details (for each of 30+ scenarios)
- **Query to Type**: Exact text to enter
- **Expected Widget**: Widget name and type
- **What the Widget Shows**: 3-4 specific data points
- **Presenter Script**: Natural language script for speaking aloud
- **Follow-Up Q&A**: 2-3 likely stakeholder questions with confident answers
- **Demo Tips**: DOs and DON'Ts for effective presentation

### 4. Cross-Persona Workflows
- Which workflows involve this persona
- Their specific role in each workflow
- Step-by-step demo flow (3-4 minutes)

### 5. Complete Demo Script (5-7 minutes total)
- Opening script (30 seconds)
- Scenario sequence (4-6 minutes)
- Closing summary (30 seconds)

### 6. Presentation Tips
- Audience-specific messaging (Executives, End Users, Technical)
- Common questions & answers (Q&A prep)
- Quick reference (top 5 queries, key metrics, demo URLs)

## üé¨ How to Use These Guides

### For Presenters (Pre-Demo)
1. **Read the full guide** for your target persona (15-20 minutes)
2. **Practice the presenter scripts** out loud - they're written for natural speech
3. **Memorize the top 5 queries** - these are your "greatest hits"
4. **Review the Q&A section** - anticipate stakeholder questions
5. **Test the queries** on the actual application to verify widget responses

### For Stakeholder Presentations
1. **Choose 3-4 scenarios** that match your audience's interests
2. **Use the opening script** to set context (30 seconds)
3. **Demo each scenario** with the provided presenter script (1-2 minutes each)
4. **Answer questions** using the prepared Q&A responses
5. **Close with impact metrics** from the guide (time savings, cost reduction)

### For Training Sessions
1. **Use the complete demo script** (5-7 minutes)
2. **Walk through all 5 scenario categories** to show breadth
3. **Demonstrate cross-persona workflows** to show collaboration
4. **Practice with trainees** - have them try the queries themselves

## üí° Tips for Success

### DO:
- ‚úÖ Practice scripts out loud before presenting - they're optimized for natural speech
- ‚úÖ Use the visual language in the guides ("red light", "heat map", "dashboard")
- ‚úÖ Connect multiple scenarios to show how issues cascade (e.g., vendor delay ‚Üí budget overrun ‚Üí resource burnout)
- ‚úÖ Emphasize time savings with specific numbers ("8 hours ‚Üí 30 seconds = 99% reduction")
- ‚úÖ Let stakeholders ask questions - the Q&A prep has you covered

### DON'T:
- ‚ùå Read scripts word-for-word robotically - use them as talking points
- ‚ùå Oversell capabilities - be honest about what V17 can and can't do
- ‚ùå Skip the opening context - stakeholders need to understand the persona's pain points
- ‚ùå Get lost in technical details unless audience requests it
- ‚ùå Rush through scenarios - give stakeholders time to absorb visualizations
